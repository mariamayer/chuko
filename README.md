# merch7am Agent Dashboard

A Next.js dashboard for managing AI agents that run on top of the merch7am FastAPI backend. Includes metrics, agent controls, run history, and multi-client management.

## How it connects to the backend

The dashboard never calls the FastAPI backend directly from the browser. Instead, `next.config.ts` sets up a rewrite:

```
/backend/* ‚Üí NEXT_PUBLIC_API_URL/*
```

So a fetch to `/backend/api/agents/performance-digest/preview` gets proxied to `http://localhost:3001/api/agents/performance-digest/preview`. This means you only need to expose one origin in production (the dashboard URL), and the backend URL never needs to be public-facing.

All API calls live in `src/lib/api.ts` and automatically attach the `NEXT_PUBLIC_API_TOKEN` as a query param (`?token=...`) to authenticate with the backend.

## Environment variables

Copy `.env.local.example` to `.env.local` and fill in:

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_API_URL` | URL of the FastAPI backend (e.g. `http://localhost:3001` or your AppRunner URL) |
| `NEXT_PUBLIC_API_TOKEN` | Must match `AGENT_TOKEN` or `REPORT_TOKEN` in the backend `.env` |
| `DASHBOARD_PASSWORD` | Single password to log into the dashboard |
| `NEXTAUTH_SECRET` | Random secret for JWT signing ‚Äî generate with `openssl rand -base64 32` |
| `NEXTAUTH_URL` | Full URL where the dashboard is running (e.g. `http://localhost:3002`) |

## Getting started locally

```bash
cd merch7am-dashboard

# 1. Install dependencies
npm install

# 2. Set up environment
cp .env.local.example .env.local
# Edit .env.local with your values

# 3. Start the dev server (port 3002)
npm run dev
```

Open [http://localhost:3002](http://localhost:3002) ‚Äî you'll be redirected to `/login`.

The FastAPI backend should be running separately on port `3001`:
```bash
cd ../merch-ai/backend
uvicorn main:app --reload --port 3001
```

## Pages

### `/dashboard` ‚Äî Overview

Loads the last 4 weeks of estimate data from the backend and displays:

- **KPI cards** ‚Äî total estimate requests, total pipeline value (‚Ç¨), average estimate value, and week-on-week trend (up / stable / down)
- **Bar charts** ‚Äî estimate count per week and pipeline value per week, rendered with Recharts
- **AI Analysis** ‚Äî the plain-English summary generated by OpenAI, refreshed each time the digest runs

Data comes from `GET /api/agents/performance-digest/preview`. No email is sent ‚Äî this is read-only.

---

### `/dashboard/agents` ‚Äî Agents

Three agent cards, one per AI agent. Each has a **Run** button that calls the backend and shows a success or error message inline.

**Performance Digest**
Calls `GET /api/agents/performance-digest/run`. The backend builds the weekly metrics, writes an AI summary, and emails it to the configured `DIGEST_EMAIL`. The run is saved to history automatically.

**SEO Content Brief**
Calls `GET /api/agents/seo-brief/all`. For every product in the Shopify catalog, the backend generates:
- 8‚Äì10 target keywords (Spanish + English)
- Meta title and meta description
- H1 heading suggestions
- Content section outline with key points
- FAQ suggestions
- Internal link anchor text ideas

**Ad Copy Refresh**
Calls `GET /api/agents/ad-copy/refresh`. For every product, the backend generates:
- 3 Meta Ads variations (headline, primary text, CTA ‚Äî within platform character limits)
- 2 Google Ads variations (3 headlines + description ‚Äî within platform character limits)
- Copy angles used (e.g. price value, quality, fast turnaround)

**Per-product targeting**
Enter a Shopify product handle (e.g. `camiseta-personalizada`) in the text field at the top. Two buttons appear ‚Äî **SEO Brief** and **Ad Copy** ‚Äî which run the agents for just that one product instead of the whole catalog.

All runs are saved to history automatically.

---

### `/dashboard/history` ‚Äî Run History

A table of every agent execution, newest first. Columns: agent name, client, status, timestamp.

**Filtering** ‚Äî use the filter buttons to show only `Performance Digest`, `SEO Brief`, or `Ad Copy` runs.

**Status values:**
- `success` ‚Äî agent ran and completed normally
- `preview` ‚Äî digest was fetched for display (no email sent)
- `error` ‚Äî agent encountered an error

**Viewing results** ‚Äî click any row to open a side drawer showing the full JSON result payload from that run. This is where you read the actual SEO briefs and ad copy generated by the agents.

Run history is stored in `merch-ai/backend/data/agent_runs/` as individual JSON files.

---

### `/dashboard/clients` ‚Äî Clients

Manage which clients the agents run for. Even with a single client, this is where credentials live.

Each client stores:
- **Name** ‚Äî display name (also used to generate the client ID slug)
- **Shopify store domain** ‚Äî e.g. `mystore.myshopify.com`
- **Shopify storefront token** ‚Äî read-only API token (masked in the UI after saving)
- **Shopify store URL** ‚Äî the public-facing URL for product links
- **Digest email** ‚Äî where the weekly digest gets sent for this client
- **Enabled modules** ‚Äî toggle which agents are active (Price Estimator, Chatbot, Performance Digest, SEO Brief, Ad Copy)

The **default** client always exists and reads from the backend's environment variables ‚Äî it cannot be deleted. Its settings are whatever is in the backend `.env` file.

Additional clients are saved as JSON files in `merch-ai/backend/data/clients/`. To run an agent for a specific client, pass `client_id` in the request body or query string.

## Authentication

Login uses [NextAuth.js](https://next-auth.js.org/) with a credentials provider. There is one shared password (`DASHBOARD_PASSWORD`). Sessions are JWT-based and last 7 days.

`src/middleware.ts` protects every route except `/login` and `/api/auth/*`. Any unauthenticated request is redirected to `/login` automatically.

To change the password, update `DASHBOARD_PASSWORD` in `.env.local` and restart the server. No database or user table is involved.

## Deployment (Vercel)

```bash
# From the merch7am-dashboard directory
git init
git add .
git commit -m "Initial commit"
# Push to a new GitHub repo, then import into Vercel
```

Set these environment variables in the Vercel project settings:

```
NEXT_PUBLIC_API_URL=https://your-backend.awsapprunner.com
NEXT_PUBLIC_API_TOKEN=your-backend-token
DASHBOARD_PASSWORD=your-secure-password
NEXTAUTH_SECRET=<output of: openssl rand -base64 32>
NEXTAUTH_URL=https://your-dashboard.vercel.app
```

The dashboard builds as a standard Next.js app ‚Äî no special Vercel configuration needed.

## Adding a new agent

To add a new AI agent to the dashboard:

1. **Backend** ‚Äî create `lib/my_agent.py` with the agent logic, `routes/my_agent.py` with the endpoints, register in `main.py`, and save runs using `lib.agent_runs.save_run()`.
2. **Dashboard** ‚Äî add a new card to `src/app/dashboard/agents/page.tsx` with a run button that calls the new endpoint via `src/lib/api.ts`. Add the agent type string to `AGENT_LABELS` in `history/page.tsx` so it displays a human-readable name in the run log.
